Welcome to Studio.

This project brings reactive programming to any javascript project. According to the reactive manifesto 2.0. Reactive systems are:

  • Responsive
  • Resilient
  • Elastic
  • Message Driven

Responsive

Studio encourages you to use actors for each "micro piece of code". Don't be worry. Studio actors always execute async, and always encapsulates your exceptions, promises and "regular object return" in a single promise, this way you will have a code a lot more mantainable and error prone.

Resilient

Studio automatically handle exceptions (expected or not) in your actors and return a simple failed promise.

Elastic

Since Studio encourages you to access your actors only through messages, it lets you deploy any number of different actors to any number of different server REALLY EASY. It let's you deploy your all application to clusters (and it will prefer to use local actors), or deploy just some of your actors to different servers. This way you can build a completely heterogenous environment easily. Forget about multiple projects, with different behaviors. Think about your application as a set of actors and redistribute as you scale (it also lets you scale back easily, why pay for processor you don't need?).

Message Driven

Javascript is a weakly typed language, so, it's perfect for Message Driven development. Using Studio's Driver, Router and Actor, you can access messages on the same way through your all application. This way, you have more mantainable and elastic code.
